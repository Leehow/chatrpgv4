# Director Orchestration Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Harden the existing CoC Story Director so rolls, failures, clue reachability, recovery, scene pacing, player proposals, and NPC agency are explicit DirectorPlan contracts instead of soft narration hints.

**Architecture:** Keep the existing deterministic chain: `coc_story_director.generate_director_plan` -> `coc_narrative_enrichment.enrich_director_plan` -> rules execution -> `coc_director_apply.backfill_rule_results` -> post-rule storylets -> `coc_director_apply.apply_plan`. Add small focused helpers around the existing files rather than a new orchestration framework.

**Tech Stack:** Python standard library, existing plugin scripts, pytest, JSON files, `scripts/sync_coc_plugin_copy.py` for Codex/ZCode synchronization.

## Global Constraints

- Canonical edits start in `plugins/coc-keeper/`; synchronize `plugins/coc-keeper-zcode/` with `python3 scripts/sync_coc_plugin_copy.py`.
- Do not branch gameplay on raw prose keywords, concrete title/name/job text, nationality, or occupation. New behavior consumes structured fields only.
- Legacy clue delivery string inference may remain compatibility-only and must be warned for critical conclusions that lack structured delivery fields.
- Director remains deterministic and read-only with respect to rule state; write effects stay in `coc_director_apply.py`.
- Every behavior change follows TDD: write the failing test, run it and verify the expected failure, implement the smallest fix, then rerun.
- Keep storylets event-triggered and anchor-bound; do not return to per-turn ambient event draws.
- Before finishing plugin work, run:
  `PYTHONDONTWRITEBYTECODE=1 python3 -m pytest tests/test_plugin_metadata.py tests/test_zcode_plugin_metadata.py tests/test_coc_plugin_sync_script.py -q -p no:cacheprovider`
  and `python3 scripts/sync_coc_plugin_copy.py --check`.

def _first_failed_contract_result(
    plan: dict[str, Any],
    rules_results: list[dict[str, Any]] | None,
) -> tuple[dict[str, Any], dict[str, Any]] | None:
    for result in rules_results or []:
        if not isinstance(result, dict):
            continue
        if _rule_result_success(result) is not False:
            continue
        contract = result.get("roll_contract")
        if not isinstance(contract, dict):
            for request in plan.get("rules_requests", []) or []:
                if not isinstance(request, dict):
                    continue
                if request.get("skill") == result.get("skill") and isinstance(request.get("roll_contract"), dict):
                    contract = request["roll_contract"]
                    break
        if isinstance(contract, dict):
            return result, contract
    return None

def _is_non_fragile_clue_route(clue: dict[str, Any]) -> bool:
    kind = clue.get("delivery_kind")
    if kind in NON_FRAGILE_DELIVERY_KINDS:
        return True
    if kind == "skill_check":
        return False
    if clue.get("fallback_route") or clue.get("recoverable") is True:

    return False


def _has_recoverable_fallback(conclusion: dict[str, Any]) -> bool:
    if conclusion.get("fallback_policy"):
        return True
    for key in ("fallback_routes", "recover_routes"):
        if conclusion.get(key):
            return True
    return any(clue.get("fallback_route") or clue.get("recoverable") is True for clue in conclusion.get("clues", []))
